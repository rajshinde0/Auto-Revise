================================================================================
                         PROJECT OVERVIEW
================================================================================

AutoRevise is a smart flashcard study application that uses spaced repetition
algorithms to help users learn and retain information effectively. The system
implements the SM-2 algorithm for optimal review scheduling and includes
gamification features to keep users engaged.

Key Features:
- User authentication and authorization
- Deck and flashcard management
- Spaced repetition learning algorithm (SM-2)
- Progress tracking and statistics
- Achievement system with badges
- CSV bulk upload for flashcards
- Study session with interactive UI
- Streak tracking for daily study habits

FILE: Backened/schema2.sql
---------------------------

Database Name: autorevise_db
Engine: InnoDB (supports foreign keys and transactions)

Tables (7):
-----------

1. Users
   - user_id (INT, PK, AUTO_INCREMENT)
   - username (VARCHAR(50), UNIQUE, NOT NULL)
   - email (VARCHAR(255), UNIQUE, NOT NULL)
   - password_hash (VARCHAR(255), NOT NULL)
   - created_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
   - points (INT, DEFAULT 0) - Gamification points

2. Decks
   - deck_id (INT, PK, AUTO_INCREMENT)
   - user_id (INT, FK → Users.user_id, CASCADE DELETE)
   - deck_name (VARCHAR(100), NOT NULL)
   - description (TEXT, NULL)
   - created_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)

3. Cards
   - card_id (INT, PK, AUTO_INCREMENT)
   - deck_id (INT, FK → Decks.deck_id, CASCADE DELETE)
   - front_content (TEXT, NOT NULL) - Question
   - back_content (TEXT, NOT NULL)  - Answer
   - created_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)

4. CardPerformance
   - performance_id (INT, PK, AUTO_INCREMENT)
   - user_id (INT, FK → Users.user_id, CASCADE DELETE)
   - card_id (INT, FK → Cards.card_id, CASCADE DELETE)
   - next_review_date (DATE, NOT NULL)
   - interval (INT, DEFAULT 1) - Days until next review
   - ease_factor (DECIMAL(4,2), DEFAULT 2.50)
   - UNIQUE KEY: (user_id, card_id)

5. StudyLog
   - log_id (INT, PK, AUTO_INCREMENT)
   - user_id (INT, FK → Users.user_id, CASCADE DELETE)
   - study_date (DATE, NOT NULL)
   - cards_reviewed (INT, DEFAULT 0)
   - UNIQUE KEY: (user_id, study_date)

6. Achievements
   - achievement_id (INT, PK, AUTO_INCREMENT)
   - name (VARCHAR(100), UNIQUE, NOT NULL)
   - description (VARCHAR(255), NOT NULL)
   - icon_url (VARCHAR(255), NULL)

7. UserAchievements
   - user_id (INT, FK → Users.user_id, CASCADE DELETE)
   - achievement_id (INT, FK → Achievements.achievement_id, CASCADE DELETE)
   - earned_at (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)
   - PRIMARY KEY: (user_id, achievement_id)

Pre-populated Achievements (6):
--------------------------------
1. First Steps          - Create your very first deck
2. Card Collector       - Create a total of 50 cards
3. Knowledge Builder    - Create a total of 250 cards
4. Dedicated Learner    - Complete your first study session
5. 7-Day Streak         - Study for 7 days in a row
6. 30-Day Streak        - Study for 30 days in a row

Relationships:
--------------
- Users → Decks (1:N)
- Decks → Cards (1:N)
- Users + Cards → CardPerformance (N:N with attributes)
- Users → StudyLog (1:N)
- Users ↔ Achievements (N:N via UserAchievements)

All foreign keys use CASCADE DELETE for automatic cleanup.

================================================================================
                    SPACED REPETITION ALGORITHM
================================================================================

Algorithm: SM-2 (SuperMemo 2)
------------------------------

The system uses the SM-2 algorithm for optimal spaced repetition:

Quality Ratings:
  0 - Complete blackout (Forgot)
  3 - Correct response with serious difficulty (Hard)
  4 - Correct response with hesitation (Good)
  5 - Perfect response (Easy)

Ease Factor (EF):
  - Initial: 2.50
  - Minimum: 1.30
  - Formula: EF = EF + (0.1 - (5-q) * (0.08 + (5-q) * 0.02))

Interval Calculation:
  - Quality < 3: Interval = 1 (restart)
  - First review: Interval = 1 day
  - Second review: Interval = 6 days
  - Subsequent: Interval = previous_interval * ease_factor

Implementation in calculate_sm2():
  Input: rating, current_interval, current_ease
  Output: new_interval, new_ease_factor

Example Flow:
  1. New card (interval=0, ease=2.5)
  2. Rate "Good" → interval=1, ease=2.5
  3. Rate "Good" → interval=6, ease=2.5
  4. Rate "Good" → interval=15, ease=2.5
  5. Rate "Hard" → interval=30, ease=2.36
  6. Rate "Easy" → interval=91, ease=2.46

Benefits:
  - Scientifically proven to improve retention
  - Adaptive to individual performance
  - Reduces study time by 90%
  - Long-term memory formation

   - No email notifications
   - No dark mode

   - No integration tests
   - No load testing
   - Manual testing only
